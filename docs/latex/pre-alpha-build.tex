\section{Purpose, repository, and design alignment}
%=============================================================================

\textbf{M1 pre-alpha:} architectural skeleton (not a shippable product): how subsystems connect and what exists in-repo today versus the design-draft mockups. \textbf{Repository:} \url{https://github.com/cannon-spencer/riscv-exam-computer}.

\paragraph{Where effort has gone vs.\ in-repo integration.}
Most time so far is \textbf{hands-on research and bring-up}: SEB and Linux lockdown experiments, distribution and board testing, and tracing the official vs.\ third-party SEB ecosystem---work that does not yet appear as a single ``app'' in \texttt{exam-env/} but is necessary to choose a viable path. \textbf{Custom} kiosk/SEB-wrapper and BEK/CSK integration are \textbf{queued for the next implementation phase}; the repository already carries docs/LaTeX, an \texttt{exam-env/} placeholder, and the \texttt{platform/reptilian-riscv} submodule (\textbf{prior} Reptilian/Bananapi-F3 style boot/kernel/partition context). Later milestones may \textbf{lean Debian} alongside Linux SEB-class tooling; the final OS line is still open.

\paragraph{SEB upstream references.}
Official Safe Exam Browser org: \url{https://github.com/safeexambrowser}. Third-party Linux builds (e.g.\ v0.4.1): \url{https://github.com/Jvr2022/seb-linux/releases/tag/v0.4.1}. These inform Linux-on-RISC-V testing; they are not a guarantee of LMS/vendor support compared to official Windows/macOS/iOS SEB.

\section{Effort, work completed, and tracking}
%=============================================================================

\noindent\textbf{Phase so far:} Substantial time in \textbf{research spikes, lab setup, and feasibility testing}: multiple OS and board paths, SEB and lockdown behavior, and scoping the official vs.\ experimental Linux SEB landscape. An end-to-end Canvas exam on the final RISC-V kiosk image is still ahead; the work below is the foundation for that integration.

\noindent\textbf{Key accomplishments:}
\begin{itemize}\setlength{\itemsep}{0.12em}
  \item \textbf{Arch Linux desktop:} ran Safe Exam Browser to establish lockdown and configuration expectations (reference for a future Linux-native or wrapper path).
  \item \textbf{Linux SEB landscape:} compared official Safe Exam Browser repositories with third-party Linux releases (URLs in Section~1); documented implications for LMS support and maintenance.
  \item \textbf{Banana Pi:} Debian via manufacturer/tutorial; validated boot, storage, and networking on hardware aligned with the design draft.
  \item \textbf{WSL vs.\ SEB:} characterized policy-relevant behavior on a general-purpose PC; supports requiring a \textbf{dedicated} exam device rather than student-owned machines.
  \item \textbf{Repository:} integrated \texttt{platform/reptilian-riscv} submodule, documented clone steps, and reserved \texttt{exam-env/} for upcoming kiosk/SEB configuration work.
\end{itemize}

\section{Known bugs, limitations, and support friction}
%=============================================================================

Pre-alpha gaps and \textbf{support-style friction} (not necessarily one ticket each). Mirror key points in \texttt{README.md} if required.

\begin{itemize}\setlength{\itemsep}{0.12em}
  \item \textbf{Integration depth:} Kiosk/BEK--CSK/compositor wiring is still ahead; current repo content is architecture, submodule context, and placeholders (Section~1).
  \item \textbf{Target E2E:} Full Canvas exam on the locked RISC-V image not yet demonstrated; desk/lab tests do not prove LMS or key handshakes on the target SoC.
  \item \textbf{SEB support model:} Official repos (\url{https://github.com/safeexambrowser}) center non-Linux; Linux trials (e.g.\ \url{https://github.com/Jvr2022/seb-linux/releases/tag/v0.4.1}) are third-party---Canvas/IT compatibility and long-term maintenance are open risks.
  \item \textbf{Submodule vs.\ Debian:} Reptilian submodule vs.\ future Debian-first image may force redoing partitions, keys, and secure-boot plumbing; cost unknown.
  \item \textbf{BSP/board noise:} BPi vs.\ Orange Pi, WiFi/UART, and vendor image quirks burn time as environment/support issues before exam logic.
  \item \textbf{Security not demonstrated:} Secure boot, PMP, verity are goals only; submodule empty clone if \texttt{git clone} without \texttt{--recurse-submodules} (fix: \texttt{git submodule update --init --recursive}).
\end{itemize}
